# v0.2.0-rc1 Release Notes (Draft)

## Highlights
- Expanded multi-chain wallet service + API maturity improvements.
- Dashboard production build validated for RC.
- Python SDK packaged and installable via editable mode.
- Added operations and governance/security documentation bundle for release readiness.

## Breaking Changes
- No confirmed protocol-level breaking API changes in this RC.
- Operational behavior may differ where upstream RPC providers are unavailable (balance endpoint can return service unavailable).

## Migration Notes
1. Ensure production secrets are set:
   - `WALLET_ENCRYPTION_KEY`
   - `API_KEY_HASH_SECRET`
   - `SHOW_BOOTSTRAP_SECRET=false`
2. Validate rate-limit strategy (`memory` vs `redis`) for your deployment topology.
3. Confirm managed RPC configuration or BYO RPC allowlist before enabling production traffic.

## Known Issues
1. Balance lookups may fail if all configured public RPC endpoints are down.
2. Python SDK repository currently has no automated pytest cases.
3. Dependency vulnerability warnings exist in npm dependency tree and should be tracked.

## Rollback Instructions
1. Identify previous stable image/revision.
2. Roll back deployment to prior revision.
3. Revert config/secret changes introduced in RC rollout.
4. Re-run smoke checks (`/health`, wallet create/read, dashboard load).

## Verification Checklist
- [ ] Service smoke tests pass in target environment
- [ ] Dashboard builds and serves static assets
- [ ] Python SDK installs successfully
- [ ] Auth paths verified (`x-api-key` enforced)
- [ ] Logging redaction checked for secret-bearing headers/query params
- [ ] Incident and rollback runbooks reviewed by on-call team
