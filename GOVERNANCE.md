# Governance

## Governance Model
Claw Wallet is maintained by the core maintainers with open community contribution.

We operate a hybrid model:
- **Community roadmap (open source):** core SDK/client capabilities, self-host runtime, and baseline documentation.
- **Commercial roadmap:** managed cloud operations, enterprise compliance modules, and hosted governance/SLA offerings.

## Decision-Making
- Maintainers are responsible for technical direction, release quality, and boundary decisions.
- Major OSS direction changes should be discussed via GitHub issues/PRs before implementation.
- In boundary disputes, maintainers will reference `OPEN_CORE_STRATEGY.md` as the source of truth.

## Community Contributions
Community members can:
- Propose features and report bugs through GitHub.
- Submit pull requests to OSS components.
- Participate in design discussions and documentation improvements.

Maintainers commit to:
- Review contributions in good faith.
- Provide clear merge/revision rationale.
- Keep OSS interfaces stable and documented.

## Roadmap Transparency
- OSS roadmap priorities are discussed publicly.
- Commercial roadmap details may remain private, but high-level categories are documented so users can understand product direction.

## Code of Conduct
All participants are expected to collaborate respectfully and constructively as described in `CONTRIBUTING.md`.
